11.5. Cài đặt các chia sẻ Windows với Samba

Samba là một tập hợp các công cụ làm việc với giao thức SMB (cũng được biết đến như là “CIFS”) trên Linux. Giao thức này được sử dụng bởi Windows cho các chia sẻ qua mạng và chia sẻ máy in.

Samba cũng có thể hoạt động như một trạm điều khiển miền máy tính (domain controller) của Windows. Nó là một công cụ tuyệt vời để đảm bảo tích hợp xuyên suốt với các máy phục vụ Linux và các máy tính bàn trong văn phòng vốn vẫn sử dụng Windows.

11.5.1. Máy phục vụ Samba

Gói phần mềm samba chứa hai máy phục vụ của Samba 4, smbd and nmbd.

TÀI LIỆU HƯỚNG DẪN Đi sâu hơn

Máy phục vụ Samba rất dễ cấu hình và đa dụng, và có thể hoạt động trong nhiều trường hợp với các yêu cầu và hạ tầng mạng ít có điểm chung. Cuốn sách này chỉ tập trung vào trường hợp Samba được sử dụng như một máy phục vụ đứng một mình, nhưng nó cũng có thể là một trạm điều khiển máy tính NT4(NT4 Domain Controller) hoặc một trạm điều khiển Active Directory(Active Directory Domain Controller), hoặc một thành viên đơn giản của một miền máy tính đã có (vốn có thể là một máy phục vụ Windows).

Gói phần mềm samba-doc chứa rất nhiều tập tin ví dụ được ghi chú trong /usr/share/doc/samba-doc/examples/.

CÔNG CỤ Xác thực với một máy phục vụ Windows

Winbind cho các nhà quản trị hệ thống một lựa chọn sử dụng máy phục vụ Windows làm máy phục vụ việc xác thực. Winbind cũng tích hợp chặt với PAM và NSS. Điều này cho phép cài đặt các máy Linux mà tất cả những người sử dụng của một miền máy tính Windows tự động nhận được một tài khoản.

Thông tin thêm có thể được tìm thấy ở thư mục /usr/share/doc/samba-doc/examples/pam_winbind/.

Mảnh thông tin bắt buộc đầu tiên là tên của nhóm làm việc(workgroup) mà máy phục vụ Samba thuộc về (câu trả lời là FALCOTNET trong trường hợp của chúng ta).

11.5.1.2.1. Những thay đổi cho smb.conf

Những yêu cầu tại Falcot bắt buộc các lựa chọn khác phải được thay đổi trong tập tin cấu hình /etc/samba/smb.conf. Đoạn mã sau tóm tắt các thay đổi trong phần [global]. 

[global]

# Thay đổi dòng sau thành nhóm-làm-việc/miền-máy-tính-NT mà máy phục vụ Samba sẽ là một phần trong đó
   workgroup = FALCOTNET

# Phần hỗ trợ Windows Internet Name Serving:
# WINS Support - Bảo thành phần NMBD của Samba để kích hoạt máy phục vụ WINS
   wins support = yes 1

[...]

11.5.1.2.2. Thêm các người dùng

Mỗi một người sử dụng Samba cần một tài khoản trên máy phục vụ; các tài khoản Unix phải được tạo trước, sau đó mỗi người sử dụng cần phải được đăng ký vào cơ sở dữ liệu của Samba. Bước Unix được thực hiện như bình thường (ví dụ, sử dụng lệnh adduser).

Thêm một người dùng đã tồn tại vào cơ sở dữ liệu Samba chỉ cần chạy lệnh smbpasswd -a user; lệnh này hỏi mật mã theo cách tương tác trực tiếp.

Một người sử dụng có thể bị xóa với lệnh smbpasswd -x user. Một tài khoản Samba có thể bị tắt đi tạm thời (với lệnh smbpasswd -d user) và được kích hoạt lại sau đó (với lệnh smbpasswd -e user).

11.5.2. Chương trình khách Samba

Các tính năng của chương trình khách trong Samba cho phép một máy tính Linux truy cập các chia sẻ và các máy in được chia sẻ của Windows. Các chương trình bắt buộc đã có sẵn trong các gói phần mềm cifs-utils và smbclient.

11.5.2.1. Chương trình smbclient

Chương trình smbclient truy xuất các máy phục vụ SMB. Nó chấp nhận lựa chọn -U user, để kết nối tới máy phục vụ dưới một danh tính cụ thể nào đó. Lệnh smbclient //server/share truy cập chia sẻ đó theo một cách có tương tác tương tự như chương trình khách FTP trên dòng lệnh. Lệnh smbclient -L server liệt kê tất cả các chia sẻ có sẵn (và nhìn thấy được) trên một máy phục vụ.

11.5.2.2. Kết nối các chia sẻ Windows

Lệnh mount cho phép kết nối một chia sẻ Windows vào một cây hệ thống tập tin Linux (với sự giúp đỡ của mount.cifs được cung cấp bởi cifs-utils).

Ví dụ 11.24. Kết nối một chia sẻ Windows

mount -t cifs //arrakis/shared /shared \
      -o credentials=/etc/smb-credentials

Tập tin /etc/smb-credentials (vốn phải không đọc được bởi người sử dụng) có định dạng sau: 

username = user
password = password

Các lựa chọn khác có thể được chỉ định trên dòng lệnh; danh sách đầy đủ của chúng có sẵn trong trang hướng dẫn sử dụng mount.cifs (1). Hai lựa chọn cụ thể có thể thú vị: uid và gid cho phép ép buộc người sở hữu và danh sách các tập tin sẽ được hiện ra trên kết nối, để khỏi phải giới hạn truy cập chỉ cho tài khoản quản trị.

Một chia sẻ Windows cũng có thể được cấu hình trong /etc/fstab: 

//server/shared /shared cifs credentials=/etc/smb-credentials

Bỏ kết nối SMB/CIFS được thực hiện với lệnh umount.

11.5.2.3. In ấn trên một máy in được chia sẻ

CUPS là một giải pháp tiện lợi cho in ấn từ một máy trạm Linux sang một máy in được chia sẻ bởi một máy Windows. Khi smbclient được cài đặt, CUPS cho phép cài đặt các máy in được chia sẻ bởi Windows một cách tự động.

Đây là các bước bắt buộc:

  • Nhập vào địa chỉ của giao diện cấu hình CUPS: http://localhost:631/admin

  • Nhấn chuộc vào “Thêm Máy In (Add Printer)”.

  • Chọn thiết bị in, chọn “Máy in Windows thông qua SAMBA“.

  • Nhập đường dẫn URI cho kết nối đến máy in mạng. Nó giống như sau:

    smb://user:password@server/printer.

  • Nhập tên vốn sẽ nhận diện được ngay máy in này. Sau đó nhập mô tả và địa điểm của máy in. Đó là những chuỗi ký tự sẽ được xuất ra cho những người dùng cuối để giúp họ nhận ra các máy in.

  • Báo nhà sản xuất/mô-đen của máy in hoặc trực tiếp cung cấp một tập tin mô tả máy in đang chạy tốt (Printer Description file - PPD).

Voilà, máy in đã hoạt động!
